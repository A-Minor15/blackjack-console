# BlackJack（コンソール版）

概要
- C++（コンソール）で実装したシンプルなブラックジャック（プレイヤー対ディーラー、1対1）。
- ルール（簡潔）
  - Aは1または11、J/Q/Kは10。
  - 合計が21を超えるとバースト（負け）。
  - 両者21以下なら合計の高い方が勝ち、同点は引き分け。
  - ディーラーは合計が17以上でスタンド。

特徴
- 表示入出力（画面更新・入力・メッセージ表示）は `Display` クラスに集中。
- ゲーム進行ロジックは `BlackJack` クラスに集約。
- カード管理は `Deck`（山札）／`Hand`（手札）クラスでカプセル化。
- Windows（MinGW）環境でのビルドを想定。

必要条件
- g++（MinGW 等）
- make（任意。PowerShell での実行を推奨する場合あり）

ビルドと実行
- Makefile（用意済み）
  make
  ./main.exe

プロジェクト構成（主要ファイル）
- include/
  - BlackJack.h
  - Display.h
  - Deck.h
  - Hand.h
- src/
  - main.cpp        — エントリポイント
  - BlackJack.cpp   — ゲームの主制御（配牌・ターン進行・判定）
  - Display.cpp     — 表示・入力の実装（画面クリア、表示更新、メッセージ、入力待ち）
  - Deck.cpp        — 山札の生成・シャッフル・配布
  - Hand.cpp        — 手札管理（合計計算、A の扱い）
- Makefile

処理の流れ（高レベル）
1. 初期化：山札を生成・シャッフル。プレイヤー／ディーラーの手札を空に。
2. 配牌フェーズ：プレイヤーとディーラーに交互に2枚ずつ配る（ディーラーの最初の1枚は表向き、2枚目を裏向きにする等の表現は Display 側で制御）。(未実装)
3. プレイヤーターン：
   - プレイヤーの合計が21未満なら `hit` / `stay` を選択。
   - `hit`：カードを引き合計を再計算。バーストならプレイヤーの敗北。
   - `stay`：プレイヤーのターン終了、ディーラーターンへ。
4. ディーラーターン：合計が17未満なら自動でヒットを繰り返す（17以上でスタンド）。
5. 勝敗判定：
   - 両者バースト → 引き分け（現在のルール）
   - 片方のみバースト → もう片方の勝ち
   - 両者21以下 → 合計が高い方が勝ち、同点は引き分け
6. 表示：すべて Display クラス経由で行う（将来的に出力先を変更しやすい）。

クラス責務
- Deck
  - 山札を生成・シャッフル・カードを配る（DrowCard 等）。
- Hand
  - 手札を保持。合計計算（A を 11 としてカウントし、21 を超える場合は 10 を引いて 1 と扱う処理）。
  - GetHands は const 参照で返す（コピーを避ける）。
- Display
  - 画面更新（ReloadConsole）、入力促進（SelectHitOrStay 等）、メッセージ表示（ShowMessage）。
  - BlackJack 側は直接 std::cout を使わず Display を呼ぶ。
- BlackJack
  - ゲームの制御（dealCards、playerTurn、dealerTurn、checkResults、ProcessBlackJack）。
